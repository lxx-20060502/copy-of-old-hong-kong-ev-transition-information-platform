How to Use This Page

  1. Keep your local EDC environment running first so the provider, consumer, and dataplane endpoints remain reachable in the background.
  2. Start the local Python backend included in this package. The live workflow on this page talks to that backend instead of relying on Postman directly.
  3. Review the prefilled inputs before running. You only need to adjust them if your local ports, URLs, or asset identifiers are different from the current setup.
  4. Click Run Transaction B. The workflow column will update in real time while the explanation card describes each governed step.
  5. The right-side response panel shows the actual request and response payloads returned from your live environment. Long JSON blocks can be scrolled inside that panel without changing the workbench layout.
  6. If anything fails, the error card will show the latest backend message so you can restart the dataspace services or recalibrate the live parameters later.
