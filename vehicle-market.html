How to Use This Page

  1. Start your local EDC environment first so the provider, consumer, and dataplane endpoints are available in the background.
  2. Run the local web server included in this package. This page talks to the Python backend directly and does not require Postman to stay open.
  3. Review the prefilled input fields before running. You only need to change them if your local ports, URLs, or asset identifiers are different.
  4. Click Run Transaction A. The workflow column will update in real time while the middle panel explains each step in plain language.
  5. The right-side response panel shows the actual request and response data returned from your live environment. Long JSON blocks can be scrolled inside that panel without changing the workbench layout.
  6. If any backend step fails, the error card will show the latest failure reason so you can adjust the URLs or restart the dataspace services and try again.
